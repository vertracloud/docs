---
title: Reset Database Certificates
sidebarTitle: Reset Certificates
api: POST https://api.vertracloud.app/v1/databases/{id}/credentials/certificate/reset
authMethod: bearer
description: Resets the certificates for a specific database by its ID.
---

{/* @see success — @vertracloud/api-types */}
<ParamField path="id" type="string" required>
  The ID of the database to reset certificates for.
</ParamField>

### Response

<ResponseField name="response" type="object">
  The new certificate details for the database.

  <Expandable title="Toggle object">

  <ResponseField name="pem" type="string">
    The new PEM certificate.
  </ResponseField>

  <ResponseField name="crt" type="string">
    The new CRT certificate.
  </ResponseField>

  <ResponseField name="key" type="string">
    The new private key for the certificate.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "response": {
    "pem": "-----BEGIN CERTIFICATE-----\nMIID...\n-----END CERTIFICATE-----",
    "crt": "-----BEGIN CERTIFICATE-----\nMIID...\n-----END CERTIFICATE-----",
    "key": "-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----"
  }
}
```

</ResponseExample>

### Error Responses

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `DATABASE_NOT_FOUND_OR_FORBIDDEN`: The specified database does not exist or the user lacks access.
  - `FAILED_TO_RESET_CERTIFICATES`: Failed to reset the certificates.
</ResponseField>

<ResponseField name="message" type="string">
  A descriptive message providing additional details about the error.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "code": "DATABASE_NOT_FOUND_OR_FORBIDDEN"
}
```

```json Error Response (500)
{
  "code": "FAILED_TO_RESET_CERTIFICATES"
}
```

</ResponseExample>
