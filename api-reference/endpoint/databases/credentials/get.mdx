---
title: Get Database Certificate
sidebarTitle: Get Certificate
api: GET https://api.vertracloud.app/v1/databases/{id}/credentials/certificate
authMethod: bearer
description: Retrieves the certificate for a specific database by its ID.
---

{/* @see binary (certificate bundle) — @vertracloud/api-types */}
<ParamField path="id" type="string" required>
  The ID of the database to retrieve the certificate for.
</ParamField>

### Response

<ResponseField name="response" type="object">
  The certificate details for the database.

  <Expandable title="Toggle object">

  <ResponseField name="pem" type="string">
    The PEM certificate.
  </ResponseField>

  <ResponseField name="crt" type="string">
    The CRT certificate.
  </ResponseField>

  <ResponseField name="key" type="string">
    The private key for the certificate.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "response": {
    "pem": "-----BEGIN CERTIFICATE-----\nMIID...\n-----END CERTIFICATE-----",
    "crt": "-----BEGIN CERTIFICATE-----\nMIID...\n-----END CERTIFICATE-----",
    "key": "-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----"
  }
}
```

</ResponseExample>

### Error Responses

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `DATABASE_NOT_FOUND_OR_FORBIDDEN`: The specified database does not exist or the user lacks access.
  - `FAILED_TO_GET_CERTIFICATE`: Failed to retrieve the certificate.
</ResponseField>

<ResponseField name="message" type="string">
  A descriptive message providing additional details about the error.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "code": "DATABASE_NOT_FOUND_OR_FORBIDDEN"
}
```

```json Error Response (500)
{
  "code": "FAILED_TO_GET_CERTIFICATE"
}
```

</ResponseExample>
