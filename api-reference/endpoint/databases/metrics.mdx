---
title: 'Get Database Metrics by ID'
sidebarTitle: 'Get Database Metrics'
api: 'GET https://api.vertracloud.app/v1/databases/{id}/metrics'
authMethod: 'bearer'
description: 'Retrieves metrics for a specific database by its ID over the last 24 hours.'
---

<ParamField path="id" type="string" required>
  The ID of the database to retrieve metrics for. If the database belongs to an organization, include the organization ID in the format `dbId-orgId`.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="array">
  An array of metric objects for the database over the last 24 hours.

  <Expandable title="Toggle object">

  <ResponseField name="cpu" type="string">
    The CPU usage of the database container (e.g., "0.00%").
  </ResponseField>

  <ResponseField name="ram" type="string">
    The RAM usage of the database container in MB (e.g., "0.00 MB").
  </ResponseField>

  <ResponseField name="storage" type="string">
    The storage usage of the database container in MB (e.g., "0.00 MB").
  </ResponseField>

  <ResponseField name="network" type="array">
    An array containing the network inbound and outbound traffic in bytes [in, out].
  </ResponseField>

  <ResponseField name="date" type="string" format="date-time">
    The timestamp when the metric was recorded.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success",
  "response": [
    {
      "cpu": "0.50%",
      "ram": "128.00 MB",
      "storage": "256.00 MB",
      "network": [1024, 2048],
      "date": "2025-08-14T19:58:00Z"
    },
    {
      "cpu": "0.75%",
      "ram": "130.00 MB",
      "storage": "256.00 MB",
      "network": [2048, 4096],
      "date": "2025-08-14T20:00:00Z"
    }
  ]
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have permission to access the database.
  - `DATABASE_NOT_FOUND`: The specified database does not exist.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "DATABASE_NOT_FOUND"
}
```

```json Error Response (401)
{
  "status": "error",
  "code": "FORBIDDEN"
}
```

</ResponseExample>
