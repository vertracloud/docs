---
title: 'Get Database Status by ID'
sidebarTitle: 'Get Database Status'
api: 'GET https://api.vertracloud.app/v1/databases/{id}/status'
authMethod: 'bearer'
description: 'Retrieves the status of a specific database by its ID.'
---

<ParamField path="id" type="string" required>
  The ID of the database to retrieve the status for. If the database belongs to an organization, include the organization ID in the format `dbId-orgId`.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="object">
  The status of the database.

  <Expandable title="Toggle object">

  <ResponseField name="cpu" type="string">
    The CPU usage of the database container (e.g., "0.00%").
  </ResponseField>

  <ResponseField name="ram" type="string">
    The RAM usage of the database container in MB (e.g., "0.00 MB").
  </ResponseField>

  <ResponseField name="storage" type="string">
    The storage usage of the database container in MB (e.g., "0.00 MB").
  </ResponseField>

  <ResponseField name="running" type="boolean">
    Indicates whether the database container is running.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success",
  "response": {
    "cpu": "1.25%",
    "ram": "128.50 MB",
    "storage": "256.00 MB",
    "running": true
  }
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `DATABASE_NOT_FOUND`: The specified database does not exist or the user lacks access.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "DATABASE_NOT_FOUND"
}
```

</ResponseExample>
