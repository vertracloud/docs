---
title: 'Reset Database'
sidebarTitle: 'Reset Database'
api: 'POST https://api.vertracloud.app/v1/databases/{id}/reset'
authMethod: 'bearer'
description: 'Resets a specific database by its ID, clearing all data.'
---

<ParamField path="id" type="string" required>
  The ID of the database to reset.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="object">
  The details of the reset database.

  <Expandable title="Toggle object">

  <ResponseField name="id" type="string">
    The ID of the reset database.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success",
  "response": {
    "id": "639d9aa35d8b4e559bcd066db4ddcc8c"
  }
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `DATABASE_NOT_FOUND`: The specified database does not exist or the user lacks access.
  - `DATABASE_ID_REQUIRED`: The database ID was not provided.
  - `DATABASE_RESET_FAILED`: Failed to reset the database.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "DATABASE_NOT_FOUND"
}
```

```json Error Response (400)
{
  "status": "error",
  "code": "DATABASE_ID_REQUIRED"
}
```

</ResponseExample>
