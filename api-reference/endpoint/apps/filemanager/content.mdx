---
title: Get Application File Content
sidebarTitle: Get File Content
api: GET https://api.vertracloud.app/v1/apps/{id}/files/content
authMethod: bearer
description: Retrieves the content of a specific file for an application by its ID.
---

{/* @see APIApplicationFileContent — @vertracloud/api-types */}
<ParamField path="id" type="string" required>
  The ID of the application to retrieve the file content from. If the application belongs to a workspace, include the workspace_id in the format `appId-workspace_id`.
</ParamField>

<ParamField query="path" type="string" required>
  The path to the file whose content is to be retrieved.
</ParamField>

### Response

<ResponseField name="response" type="object">
  The content of the requested file.

  <Expandable title="Toggle object">

  <ResponseField name="data" type="array">
    An array of numbers representing the file content in bytes.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "response": {
    "data": [72, 101, 108, 108, 111, 32, 87, 111, 114, 108, 100]
  }
}
```

</ResponseExample>

### Error Responses

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have permission to access the application.
  - `APP_NOT_FOUND`: The specified application does not exist.
  - `PLAN_RESTRICTED_FEATURE`: The user's plan does not support file manager access (except for `vertracloud.config`).
  - `MISSING_PATH`: The path query parameter was not provided.
  - `FILE_CONTENT_ERROR`: Failed to retrieve the file content.
</ResponseField>

<ResponseField name="message" type="string">
  A descriptive message providing additional details about the error.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "code": "APP_NOT_FOUND"
}
```

```json Error Response (400)
{
  "code": "MISSING_PATH",
  "message": "Path query parameter is required"
}
```

</ResponseExample>
