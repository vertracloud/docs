---
title: 'Move Application File or Folder'
sidebarTitle: 'Move File or Folder'
api: 'PUT https://api.vertracloud.app/v1/apps/{id}/files/move'
authMethod: 'bearer'
description: 'Moves a file or folder within an application to a new location by its ID.'
---

<ParamField path="id" type="string" required>
  The ID of the application to move the file or folder within. If the application belongs to an organization, include the organization ID in the format `appId-orgId`.
</ParamField>

<ParamField query="path" type="string" required>
  The source path of the file or folder to move.
</ParamField>

<ParamField query="to" type="string" required>
  The destination path to move the file or folder to.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success"
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have permission to access the application.
  - `APP_NOT_FOUND`: The specified application does not exist.
  - `PLAN_RESTRICTED_FEATURE`: The user's plan does not support file manager access.
  - `MISSING_PATH_OR_DESTINATION`: The path or destination query parameter was not provided.
  - `FORBIDDEN_PATH`: The specified source path is restricted.
  - `FILE_MOVE_FAILED`: Failed to move the file or folder.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "APP_NOT_FOUND"
}
```

```json Error Response (400)
{
  "status": "error",
  "code": "MISSING_PATH_OR_DESTINATION",
  "message": "Both 'path' and 'to' query parameters are required"
}
```

</ResponseExample>
