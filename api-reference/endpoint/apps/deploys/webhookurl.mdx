---
title: 'Get Deploy Webhook URL'
sidebarTitle: 'Get Webhook URL'
api: 'GET https://api.vertracloud.app/v1/apps/{app_id}/deploys/webhook'
authMethod: 'bearer'
description: 'Retrieves the GitHub deploy webhook URL and repository information for an application.'
---

<ParamField path="app_id" type="string" required>
The ID of the application whose webhook URL will be retrieved.
</ParamField>

### Response

<ResponseField name="status" type="string">
Indicates if the request was successful. Returns `success` on success.
</ResponseField>

<ResponseField name="response" type="object">
Contains repository and webhook details.

<Expandable title="Toggle object">
  <ResponseField name="repo_owner" type="string">
  The owner of the GitHub repository.
  </ResponseField>

  <ResponseField name="repo_name" type="string">
  The name of the GitHub repository.
  </ResponseField>

  <ResponseField name="webhook_url" type="string">
  The URL of the deploy webhook.
  </ResponseField>
</Expandable>
</ResponseField>

<ResponseExample>
```json
{
  "status": "success",
  "response": {
    "repo_owner": "owner_name",
    "repo_name": "repository_name",
    "webhook_url": "https://api.vertracloud.app/v1/apps/123/deploys/receiver/webhook"
  }
}
```
</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
Returns `error` if the request fails.
</ResponseField>

<ResponseField name="code" type="string">
Possible error codes:
- `UNAUTHORIZED`: User is not authenticated or lacks permissions.
- `PLAN_RESTRICTED_FEATURE`: User plan does not support auto-deploy.
- `DEPLOYMENT_NOT_FOUND`: Webhook integration not found for the application.
</ResponseField>

<ResponseExample>
```json
{
  "status": "error",
  "code": "DEPLOYMENT_NOT_FOUND"
}
```
</ResponseExample>
