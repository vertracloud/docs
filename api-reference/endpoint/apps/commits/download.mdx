---
title: 'Download Application Commit'
sidebarTitle: 'Download Commit'
api: 'GET https://api.vertracloud.app/v1/apps/:id/commits/:commit_id/download'
authMethod: 'bearer'
description: 'Downloads the ZIP file of a specific commit for a given application.'
---

<ParamField path="id" type="string">
  The ID of the application.
</ParamField>

<ParamField path="commit_id" type="string">
  The ID of the commit to download.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the operation was successful. `success` if the commit ZIP was retrieved.
</ResponseField>

<ResponseField name="response" type="file">
  The ZIP file containing the application at the specified commit.
</ResponseField>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the operation was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have access to the application.
  - `APP_NOT_FOUND`: The specified application does not exist.
  - `COMMIT_NOT_FOUND`: The specified commit does not exist.
  - `COMMIT_DATA_NOT_FOUND`: The commit data could not be retrieved from the server.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (403)
{
  "status": "error",
  "code": "FORBIDDEN"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "APP_NOT_FOUND"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "COMMIT_NOT_FOUND"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "COMMIT_DATA_NOT_FOUND"
}
```

</ResponseExample>
