---
title: 'Get Application Commits'
sidebarTitle: 'Application Commits'
api: 'GET https://api.vertracloud.app/v1/apps/:id/commits'
authMethod: 'bearer'
description: 'Retrieves a list of commits for a specified application.'
---

<ParamField path="id" type="string">
  The ID of the application to retrieve commits for.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the operation was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="array">
  An array of commit objects.

  <Expandable title="Toggle object">

  <ResponseField name="id" type="string">
    The commit ID.
  </ResponseField>

  <ResponseField name="author_id" type="string">
    The ID of the user who authored the commit.
  </ResponseField>

  <ResponseField name="message" type="string">
    The commit message.
  </ResponseField>

  <ResponseField name="version" type="string">
    The version associated with the commit.
  </ResponseField>

  <ResponseField name="status" type="string">
    The status of the commit.
  </ResponseField>

  <ResponseField name="size" type="string">
    The size of the commit in MB.
  </ResponseField>

  <ResponseField name="date" type="string">
    The timestamp when the commit was created.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success",
  "response": [
    {
      "id": "commit123",
      "author_id": "user123",
      "message": "Initial commit",
      "version": "1.0.0",
      "status": "success",
      "size": "1.23 MB",
      "date": "2025-08-14T21:00:00.000Z"
    }
  ]
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the operation was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have access to the application.
  - `APP_NOT_FOUND`: The specified application does not exist.
</ResponseField>

<ResponseExample>

```json Error Response (401)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (403)
{
  "status": "error",
  "code": "FORBIDDEN"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "APP_NOT_FOUND"
}
```

</ResponseExample>
