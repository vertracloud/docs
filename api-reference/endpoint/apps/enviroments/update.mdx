---
title: 'Update Application Environment'
sidebarTitle: 'Update Environment'
api: 'PUT https://api.vertracloud.app/v1/apps/{id}/envs'
authMethod: 'bearer'
description: 'Updates a specific environment variable for an application by its ID.'
---

<ParamField path="id" type="string" required>
  The ID of the application to update the environment variable for. If the application belongs to an organization, include the organization ID in the format `appId-orgId`.
</ParamField>

<ParamField body="environment" type="object" required>
  The environment variable to update.

  <Expandable title="Toggle object">

  <ParamField name="id" type="string" required>
    The ID of the environment variable to update.
  </ParamField>

  <ParamField name="key" type="string" required>
    The new key of the environment variable (max 100 characters).
  </ParamField>

  <ParamField name="value" type="string" required>
    The new value of the environment variable (max 1000 characters).
  </ParamField>

  <ParamField name="note" type="string">
    An optional updated note or description for the environment variable.
  </ParamField>

  </Expandable>
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="array">
  An array of all environment variable objects after the update.

  <Expandable title="Toggle object">

  <ResponseField name="id" type="string">
    The ID of the environment variable.
  </ResponseField>

  <ResponseField name="key" type="string">
    The key of the environment variable.
  </ResponseField>

  <ResponseField name="value" type="string">
    The value of the environment variable (decrypted).
  </ResponseField>

  <ResponseField name="note" type="string | null">
    The optional note or description for the environment variable.
  </ResponseField>

  <ResponseField name="created_at" type="string" format="date-time">
    The timestamp when the environment variable was created.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success",
  "response": [
    {
      "id": "env123",
      "key": "API_KEY",
      "value": "newsecret456",
      "note": "Updated API key for external service",
      "created_at": "2025-08-14T19:58:00Z"
    },
    {
      "id": "env456",
      "key": "DATABASE_URL",
      "value": "mysql://user:pass@localhost/db",
      "note": null,
      "created_at": "2025-08-14T19:58:00Z"
    }
  ]
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have permission to access the application.
  - `APP_NOT_FOUND`: The specified application does not exist.
  - `INVALID_ENVIRONMENT`: The provided environment variable is invalid or uses a forbidden key.
  - `ENV_VARIABLE_TOO_LONG`: The environment variable key or value exceeds length limits.
  - `ENVIRONMENT_NOT_FOUND`: The specified environment variable ID was not found.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "APP_NOT_FOUND"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "ENVIRONMENT_NOT_FOUND"
}
```

</ResponseExample>
