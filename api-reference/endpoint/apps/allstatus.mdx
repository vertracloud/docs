---
title: 'All Applications Status'
sidebarTitle: 'All Applications Status'
api: 'GET https://api.vertracloud.app/v1/apps/status'
authMethod: 'bearer'
description: 'Retrieves the status of all applications for the authenticated user or a specified organization.'
---

<ParamField query="organization_id" type="string">
  The ID of the organization to retrieve application statuses for, if applicable.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="array">
  An array of status objects for each application.

  <Expandable title="Toggle object">

  <ResponseField name="id" type="string">
    The applicationâ€™s ID.
  </ResponseField>

  <ResponseField name="cpu" type="string">
    The CPU usage percentage (e.g., `0%`).
  </ResponseField>

  <ResponseField name="ram" type="string">
    The RAM usage (e.g., `0 MB`).
  </ResponseField>

  <ResponseField name="running" type="boolean">
    Indicates whether the application is currently running.
  </ResponseField>

  </Expandable>
</ResponseField>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `ORGANIZATION_NOT_FOUND`: The specified organization does not exist.
  - `USER_NOT_IN_ORGANIZATION`: The user is not a member of the specified organization.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "ORGANIZATION_NOT_FOUND"
}
```

```json Error Response (403)
{
  "status": "error",
  "code": "USER_NOT_IN_ORGANIZATION"
}
```

</ResponseExample>
