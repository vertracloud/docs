---
title: 'Create Application Backup'
sidebarTitle: 'Create Application Backup'
api: 'POST https://api.vertracloud.app/v1/apps/{id}/backups'
authMethod: 'bearer'
description: 'Creates a backup for an application by its ID.'
---

<ParamField path="id" type="string" required>
  The ID of the application to create a backup for. If the application belongs to an organization, include the organization ID in the format `appId-orgId`.
</ParamField>

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="object">
  Details of the created backup.

  <Expandable title="Toggle object">

  <ResponseField name="id" type="string">
    The ID of the backup.
  </ResponseField>

  <ResponseField name="app_id" type="string">
    The ID of the application.
  </ResponseField>

  <ResponseField name="author_id" type="string">
    The ID of the user who created the backup.
  </ResponseField>

  <ResponseField name="date" type="string" format="date-time">
    The timestamp when the backup was created.
  </ResponseField>

  <ResponseField name="size" type="string">
    The size of the backup in MB.
  </ResponseField>

  </Expandable>
</ResponseField>

<ResponseExample>

```json Response
{
  "status": "success",
  "response": {
    "id": "backup123",
    "app_id": "v111d4af8cd74272a284831fbae14a8v",
    "author_id": "1085944713435160717",
    "date": "2025-08-14T19:58:00Z",
    "size": "10.24 MB"
  }
}
```

</ResponseExample>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
  - `FORBIDDEN`: The user does not have permission to access the application.
  - `APP_NOT_FOUND`: The specified application does not exist.
  - `PLAN_RESTRICTED_FEATURE`: The user's plan does not support creating backups.
  - `BACKUP_FAILED`: The backup creation failed.
</ResponseField>

<ResponseExample>

```json Error Response (403)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

```json Error Response (404)
{
  "status": "error",
  "code": "APP_NOT_FOUND"
}
```

```json Error Response (500)
{
  "status": "error",
  "code": "BACKUP_FAILED"
}
```

</ResponseExample>
