---
title: 'All Application Backups'
sidebarTitle: 'All Application Backups'
api: 'GET https://api.vertracloud.app/v1/apps/backups'
authMethod: 'bearer'
description: 'Retrieves all backups for applications owned by the authenticated user, grouped by application.'
---

### Response

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `success` if successful, `error` if not.
</ResponseField>

<ResponseField name="response" type="array">
  An array of objects, each representing an application and its associated backups.

  <Expandable title="Toggle object">

  <ResponseField name="app_id" type="string">
    The ID of the application.
  </ResponseField>

  <ResponseField name="app_name" type="string">
    The name of the application (defaults to app_id if name is unavailable).
  </ResponseField>

  <ResponseField name="backups" type="array">
    An array of backup objects for the application.

    <Expandable title="Toggle object">

    <ResponseField name="id" type="string">
      The ID of the backup.
    </ResponseField>

    <ResponseField name="app_id" type="string">
      The ID of the application.
    </ResponseField>

    <ResponseField name="author_id" type="string">
      The ID of the user who created the backup.
    </ResponseField>

    <ResponseField name="size" type="string">
      The size of the backup in MB.
    </ResponseField>

    <ResponseField name="date" type="string" format="date-time">
      The timestamp when the backup was created.
    </ResponseField>

    </Expandable>
  </ResponseField>

  </Expandable>
</ResponseField>

### Error Responses

<ResponseField name="status" type="string">
  Indicates whether the call was successful. `error` if not successful.
</ResponseField>

<ResponseField name="code" type="string">
  The error code indicating the reason for failure. Possible values:
  - `UNAUTHORIZED`: The user is not authenticated or lacks sufficient permissions.
</ResponseField>

<ResponseExample>

```json Error Response (401)
{
  "status": "error",
  "code": "UNAUTHORIZED"
}
```

</ResponseExample>
