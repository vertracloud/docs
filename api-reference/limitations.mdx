## Bandwidth Limits

Network bandwidth is allocated proportionally to the memory assigned to each resource.

| RAM | Bandwidth |
|-----|:---------:|
| ≤ 256 MB | 50 Mbit |
| ≤ 512 MB | 100 Mbit |
| ≤ 1,024 MB | 200 Mbit |
| ≤ 2,048 MB | 400 Mbit |
| ≤ 4,096 MB | 800 Mbit |
| ≤ 8,192 MB | 1,600 Mbit |
| ≤ 10,240 MB | 2,000 Mbit |

Resources exceeding 10 GB of RAM receive bandwidth scaled at approximately 50 Mbit per 256 MB.

## Upload Limits

| Resource | Limit |
|----------|:-----:|
| ZIP file size | 100 MB |
| Build time | 5 minutes |
| Exposed ports | 1 per application |

## Auto-Restart

Applications with auto-restart enabled are subject to crash loop detection. If a container crashes **5 times in 10 minutes**, auto-restart is disabled for 24 hours. There is a **1-hour cooldown** between restarts, and the container must be online for at least **60 seconds** to count as stable.

## Free Plan

Projects on the Free plan that remain offline for more than **30 days** are automatically deleted. Notifications are sent at days 7, 23 and 27.
