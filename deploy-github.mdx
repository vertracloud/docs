## GitHub Integration

Vertra Cloud offers direct integration with GitHub to facilitate the deployment process. You can select a repository (including private repositories) during project creation and configure webhooks for automatic deployment.

## Manual Deploy via GitHub

<Steps>
  <Step title="Create New Project">
    In the dashboard, click "New Project" and select "Application".
  </Step>
  <Step title="Select Source: GitHub">
    In the source selection step, choose "GitHub" and authorize access to your account.
  </Step>
  <Step title="Choose Repository">
    Select the desired repository from the list. Private repositories are also supported.
  </Step>
  <Step title="Select Main File">
    Choose the application's main file (ex: `index.js`, `main.py`).
  </Step>
  <Step title="Configure Resources">
    Define memory, storage and billing mode (fixed plan or credits).
  </Step>
  <Step title="Review and Create">
    Review the settings and click "Create Application".
  </Step>
</Steps>

## Automatic Deploy with Webhooks

After creating the application, you can configure a GitHub deploy webhook so that every time a push is made to the configured branch, the application is automatically updated.

### Configure Webhook

1. In the application settings, go to the "Deploys" tab
2. Copy the webhook URL provided by the platform
3. In GitHub, go to **Settings → Webhooks → Add webhook**
4. Paste the URL and configure it to trigger on push events

<Note>
  Each application has a unique webhook URL. Do not share this URL publicly.
</Note>

## Deploy via ZIP

If you prefer not to use GitHub integration, you can upload a ZIP file directly. The creation wizard offers a drag-and-drop area to facilitate the process.

**Before creating the ZIP, exclude:**
- `node_modules`, `.npm`, `package-lock.json` (JavaScript/TypeScript)
- `venv`, `.cache`, `__pycache__` (Python)
- `target` (Rust/Java)
- `vendor` (PHP/Ruby)
